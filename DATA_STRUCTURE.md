# Структура данных проекта bank-products-static

## Обновленная структура хранения

Согласно концепции версионирования данных, все JSON файлы банковских продуктов теперь организованы следующим образом:

```
src/
└── data/
    └── pdfs/                           # Основные банковские продукты
        ├── credit-cards/              # Кредитные карты
        │   ├── [bank-name-latin]/    # Продукты конкретного банка (латинские символы)
        │   │   └── [bank-name]-[product-name]-[YYYY-MM-DD].json
        │   └── ...
        ├── debit-cards/               # Дебетовые карты
        │   └── [bank-name-latin]/    # Продукты конкретного банка (латинские символы)
        │       └── [bank-name]-[product-name]-[YYYY-MM-DD].json
        ├── credits/                   # Кредиты
        │   └── [bank-name-latin]/    # Продукты конкретного банка (латинские символы)
        │       └── [bank-name]-[product-name]-[YYYY-MM-DD].json
        ├── deposits/                  # Вклады
        │   └── [bank-name-latin]/    # Продукты конкретного банка (латинские символы)
        │       └── [bank-name]-[product-name]-[YYYY-MM-DD].json
        └── ...
```

## Структура JSON файла продукта

Каждый банковский продукт представлен в виде JSON файла со следующей обязательной структурой:

```json
{
  "id": "уникальный-идентификатор-продукта",
  "type": "тип-продукта",
  "bankName": "название-банка",
  "title": "название-продукта",
  "version": {
    "date": "YYYY-MM-DD",
    "source": "источник-информации",
    "updatedBy": "автор-изменений"
  },
  "parameters": {
    "main": {
      "ключевой-параметр": "значение"
    },
    "fees": {
      "комиссия": "значение"
    },
    "requirements": {
      "требование": "значение"
    }
  },
  "validFrom": "YYYY-MM-DD",
  "status": "active|discontinued|updated",
  "featured": false,
  "shortDescription": "краткое-описание",
  "fullDescription": "полное-описание",
  "imageUrl": "https://bank-select.ru/images/...",
  "features": ["особенность1", "особенность2"],
  "conditions": ["условие1", "условие2"],
  "referralLink": "https://...",
  "meta": {
    "title": "SEO заголовок",
    "description": "SEO описание"
  }
}
```

## Именование файлов

Файлы именуются по следующему шаблону:
```
[bank-name]-[product-name]-[YYYY-MM-DD].json
```

Где:
- `[bank-name]` - нормализованное название банка (в нижнем регистре, с дефисами вместо пробелов и специальных символов, с транслитерацией кириллических символов в латиницу)
- `[product-name]` - нормализованное название продукта (в нижнем регистре, с дефисами вместо пробелов и специальных символов, с транслитерацией кириллических символов в латиницу)
- `[YYYY-MM-DD]` - дата последнего обновления в формате год-месяц-день

Для транслитерации кириллических символов в латиницу используется следующая таблица:
- а → a, б → b, в → v, г → g, д → d, е → e, ё → yo
- ж → zh, з → z, и → i, й → y, к → k, л → l, м → m
- н → n, о → o, п → p, р → r, с → s, т → t, у → u
- ф → f, х → kh, ц → ts, ч → ch, ш → sh, щ → shch
- ъ → (пусто), ы → y, ь → (пусто), э → e, ю → yu, я → ya

## Требования к структуре

Все JSON файлы банковских продуктов должны соответствовать схеме, определенной в `src/schemas/product-schema.json`.

Для проверки соответствия можно использовать команду:
```bash
npm run validate
```

## Миграция данных

Для миграции существующих данных к новой структуре использовался скрипт `src/scripts/migrate-data-structure.js`, который:
1. Добавляет недостающие обязательные поля
2. Обновляет формат URL изображений до полного URI
3. Устанавливает значения по умолчанию для опциональных полей

Для переименования и перемещения файлов в соответствии с новой структурой директорий использовался скрипт `src/scripts/restructure-data.js`.

## Версионирование

Каждый продукт имеет:
- Дату начала действия (`validFrom`)
- Статус (`active`, `discontinued`, `updated`)
- Информацию о версии (`version.date`, `version.source`, `version.updatedBy`)

Для отслеживания изменений используется Git.
