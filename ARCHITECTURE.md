# Архитектура проекта bank-products-static

## Общая концепция

Проект представляет собой статический сайт, генерируемый из JSON файлов с информацией о банковских продуктах. Архитектура основана на принципах простоты, надежности и прозрачности. Управление контентом осуществляется через Git, а не через CMS.

## Структура проекта

```
src/
├── data/                 # JSON файлы с информацией о продуктах
│   ├── pdfs/             # Основные продукты
│   │   ├── credit-cards/ # Кредитные карты
│   │   ├── debit-cards/  # Дебетовые карты
│   │   ├── credits/      # Кредиты
│   │   └── [bank-name]/  # Продукты конкретного банка
│   ├── banks.json        # Информация о банках
│   └── categories.json   # Категории продуктов
├── img/                  # Изображения
│   └── bank-logos/       # Логотипы банков
├── scripts/              # Скрипты генерации
│   ├── generate.js       # Основной скрипт генерации
│   ├── validate.js       # Скрипт валидации
│   └── llm-validate.js   # Скрипт валидации через LLM
├── static/               # Статические файлы
├── styles/               # Стили
└── templates/            # Шаблоны Handlebars
    ├── articles/         # Статьи
    ├── layout.html       # Основной макет
    ├── product-page.html # Страница продукта
    ├── category-page.html # Страница категории
    ├── compare-page.html # Страница сравнения
    └── index-page.html   # Главная страница
```

## Pipeline генерации

### 1. Сбор данных
- JSON файлы с информацией о продуктах хранятся в `src/data/pdfs/`
- Файлы группируются по типам продуктов и банкам
- Используется версионирование через имена файлов и директории

### 2. Валидация
- Проверка структуры JSON файлов
- Проверка соответствия схеме
- Проверка через LLM (актуальность, достоверность)
- Автоматическая проверка в Git workflow

### 3. Генерация
- Скрипт `generate.js` читает все JSON файлы
- Использует шаблоны Handlebars для генерации HTML
- Создает статические страницы для продуктов, категорий и сравнений
- Копирует статические ресурсы

### 4. Публикация
- Сгенерированные файлы помещаются в директорию `dist/`
- Публикация на GitHub Pages

## Использование LLM в pipeline

### 1. Валидация актуальности
- Периодическая проверка параметров продукта
- Сравнение с официальной информацией на сайтах банков
- Определение необходимости обновления

### 2. Проверка достоверности
- Проверка логической согласованности параметров
- Определение потенциальных ошибок
- Оценка надежности источника информации

### 3. Автоматическое форматирование
- Генерация описаний и метаданных
- Проверка структуры JSON файлов
- Корректировка форматирования

### 4. Обнаружение дубликатов
- Сравнение новых продуктов с существующими
- Определение степени схожести
- Предотвращение дублирования информации

## Git workflow

### 1. Добавление новых продуктов
- Создание JSON файла с информацией о продукте
- Проверка через автоматические тесты
- Проверка через LLM
- Создание pull request
- Рецензирование и слияние

### 2. Обновление существующих продуктов
- Изменение соответствующего JSON файла
- Указание причины изменения
- Проверка через LLM
- Создание pull request

### 3. Автоматические проверки
- Валидация JSON структуры
- Проверка через LLM
- Тестирование генерации
- SEO проверка
